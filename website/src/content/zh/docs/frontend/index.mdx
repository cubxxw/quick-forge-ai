# 前端开发

Quick Forge AI 前端使用 TypeScript 和 React 构建，提供现代、类型安全的用户界面。本节涵盖前端架构、关键组件以及如何根据您的需求扩展前端。

## 架构概述

前端使用 React 遵循基于组件的架构：

1. **组件** - 可重用的 UI 构建块
2. **页面** - 由组件构建的路由特定视图
3. **钩子** - 用于状态管理和副作用的自定义逻辑
4. **服务** - API 客户端和外部集成
5. **实用工具** - 辅助函数和共享代码

## 关键技术

- **TypeScript**：JavaScript 的类型安全
- **React**：UI 组件库
- **Chakra UI**：响应式设计的组件库
- **React Router**：客户端路由
- **React Query**：数据获取和缓存
- **Vite**：快速构建工具和开发服务器
- **Jest/Testing Library**：测试框架

## 目录结构

```
frontend/
├── src/
│   ├── components/     # 可重用的 UI 组件
│   │   ├── common/     # 共享组件
│   │   ├── layout/     # 布局组件
│   │   └── forms/      # 表单组件
│   ├── hooks/          # 自定义 React 钩子
│   ├── pages/          # 页面组件
│   ├── services/       # API 客户端和服务
│   │   ├── api.ts      # 自动生成的 API 客户端
│   │   └── auth.ts     # 认证服务
│   ├── types/          # TypeScript 类型定义
│   ├── utils/          # 实用函数
│   ├── App.tsx         # 根组件
│   └── main.tsx        # 入口点
├── tests/              # 测试文件
│   ├── components/     # 组件测试
│   └── utils/          # 实用工具测试
├── public/             # 静态资源
└── vite.config.ts      # Vite 配置
```

## 开始前端开发

要开始开发前端：

1. **了解现有代码**：
   - 探索 `src/components/` 中的组件结构
   - 查看 `src/pages/` 中的页面组件
   - 查看 `src/services/` 中的 API 集成

2. **设置开发环境**：
   - 按照[设置说明](/zh/docs/getting-started)操作
   - 运行前端开发服务器：`cd frontend && pnpm dev`

3. **访问应用程序**：
   - 运行后，在 `http://localhost:3000` 访问应用程序
   - 后端也应该运行以使 API 调用正常工作

## 主要组件

### UI 组件

UI 由 `src/components/` 中组织的可重用组件构建。这些包括：

- **通用组件**：按钮、卡片、模态框等
- **布局组件**：页面布局、导航、侧边栏
- **表单组件**：输入字段、表单、验证
- **功能组件**：与特定功能相关联的组件

### 路由

路由使用 `src/App.tsx` 中的 React Router 处理，定义：

- 公共路由（任何人都可访问）
- 受保护路由（需要认证）
- 布局路由（用于一致的页面结构）

### 状态管理

状态使用以下组合管理：

- React 内置的 `useState` 和 `useReducer` 用于本地状态
- Context API 用于组件间共享状态
- React Query 用于服务器状态（API 数据）

### API 集成

前端通过以下方式与后端通信：

- 基于 OpenAPI 架构的自动生成 TypeScript API 客户端
- `src/services/` 中的自定义服务模块
- 使用 JWT 令牌的认证处理

### 主题

应用程序使用 Chakra UI 的主题系统支持深色/浅色模式，具有：

- 一致的配色方案
- 适用于所有屏幕尺寸的响应式设计
- 遵循 WCAG 指南的无障碍组件
